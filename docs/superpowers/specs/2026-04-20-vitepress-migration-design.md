# VitePress 迁移设计（Notes-Atlas）

## 1. 目标与范围

将当前 Docsify 静态站点迁移到 VitePress，保持现有 Markdown 内容目录基本不变，完成可构建、可部署、可浏览的单站点形态，并迁移关键功能。

### In Scope

- 使用 VitePress 重建站点配置、导航与主题扩展。
- 保留现有内容目录（`meet/`、`study/`、`linux-install/`、`others/` 等）并可直接访问。
- 迁移核心能力：搜索、数学公式渲染、代码复制、更新时间提示、导出 PDF/HTML 按钮。
- `/others` 路径保留“受保护”语义，实际鉴权由反向代理/网关实现。

### Out of Scope

- 重写全部历史文档内容。
- 在应用层实现真正的密码鉴权（由部署层负责）。

## 2. 架构设计

### 2.1 目录与构建

- 新增 `package.json`（`docs:dev` / `docs:build` / `docs:preview`）。
- 新增 `.vitepress/`：
  - `config.mts`：站点配置、导航、侧边栏、主题配置。
  - `theme/index.ts`：主题增强入口。
  - `theme/components/`：更新时间与导出控件组件。
  - `theme/utils/`：导出、更新时间读取等工具函数。
- 将 `static/` 资源迁入 `public/`（或保持兼容映射），确保资源 URL 稳定。

### 2.2 路由与链接策略

- 根路径部署（`base: "/"`），后续如切到子路径部署仅改 `base`。
- 保持当前文档路径风格，尽量不改正文链接。
- 对历史 Docsify 风格路由在导航层做兼容映射，避免常用入口失效。

### 2.3 导航迁移

- 以现有 `navbar.md`、`_sidebar.md` 为来源，迁入 `themeConfig.nav` 与 `themeConfig.sidebar`。
- 一级栏目保持：论文笔记、学习笔记、Linux 安装、其他资源。

## 3. 功能设计

### 3.1 搜索与基础体验

- 采用 VitePress 本地搜索能力（默认）；
- 使用 VitePress 内置代码复制；
- 通过 Markdown/主题配置启用 KaTeX 或等效数学渲染能力。

### 3.2 最新更新时间

- 继续使用现有脚本 `scripts/generate-last-updated.sh` 产出更新时间 JSON。
- JSON 文件放入 `public/last-updated.json`（或兼容原路径）。
- 在主题组件中按当前页面路径查表并展示“最新更新时间: YYYY/MM/DD”。
- 文件缺失或 key 不存在时明确不展示提示，不进行误导性默认值回退。

### 3.3 导出 PDF/HTML

- 在主题中增加“导出”按钮（仅文章页显示）。
- HTML 导出：抓取主内容容器，拼接最小 HTML 文档并触发下载。
- PDF 导出：使用 `html2canvas + jsPDF`，按 A4 分页生成并下载。
- 导出失败时给出可见错误提示。

### 3.4 `/others` 保护语义

- VitePress 内保留该路径入口与提示文案（受保护内容）。
- 实际访问控制由 Nginx/Caddy/网关实现（Basic Auth、SSO 或其他策略）。

## 4. 数据流与交互

1. 页面渲染完成后，更新时间组件读取路由 -> 拉取 `last-updated.json` -> 显示日期标签。
2. 用户点击导出按钮：
   - HTML：读取当前内容 DOM -> 生成文档字符串 -> 下载。
   - PDF：克隆内容节点 -> 等待图片/字体 -> 画布渲染 -> PDF 分页输出。

## 5. 错误处理策略

- 构建期：对导航路径与常用入口做链接可达检查（避免迁移后 404）。
- 运行期：
  - 更新时间 JSON 加载失败：显示“更新时间不可用”或不显示标签（按设计统一）。
  - PDF/HTML 导出依赖未加载或渲染失败：显示明确错误消息。
- 不使用吞错式 silent failure，保证用户可感知失败原因。

## 6. 验收标准

- 可执行 `docs:dev` 启动本地站点，`docs:build` 成功产物可预览。
- 首页与主要一级目录可访问。
- 数学公式、代码复制、搜索可用。
- 更新时间提示可按页面显示。
- 导出 PDF/HTML 在典型页面可用。
- `/others` 页面具备受保护语义提示，并预留部署层鉴权接入点。

## 7. 风险与缓解

- **风险：** Docsify 插件行为与 VitePress 机制差异大。  
  **缓解：** 将行为下沉到主题组件与工具函数，减少全局脚本注入。
- **风险：** 历史链接可能不一致导致断链。  
  **缓解：** 导航映射 + 迁移后抽样检查核心入口。
- **风险：** 导出功能依赖 DOM 结构，主题改动可能影响导出。  
  **缓解：** 固定导出目标容器选择器并集中维护。
