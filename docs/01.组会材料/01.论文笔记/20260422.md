# 2026-04-22组会内容

## Fabrica: Dual-Arm Assembly of General Multi-Part Objects via Integrated Planning and Learning

- Project Link：[https://fabrica.csail.mit.edu/](https://fabrica.csail.mit.edu/)
- Github Link: [https://github.com/yunshengtian/Fabrica](https://github.com/yunshengtian/Fabrica)
- Paper Link：[https://arxiv.org/pdf/2506.05168](https://arxiv.org/pdf/2506.05168)

### 摘要

多部件装配对机器人执行长视野、接触丰富的操作以及跨复杂几何形状的泛化提出了重大挑战。本工作提出 Fabrica，一种面向通用多部件物体的双臂装配系统，实现了从全局规划到局部控制的端到端集成。

在规划层面，作者构建了一个分层结构，联合优化装配优先级、顺序、抓取以及运动规划，并引入自动夹具生成模块，使系统能够在不同双臂机器人平台上执行多步骤装配任务。该规划器通过可并行设计与离线预计算机制提升效率，同时保证下游控制的稳定性。

在控制层面，针对接触密集的插入操作，提出了一种轻量级残差强化学习框架，在规划轨迹的引导下学习鲁棒控制策略，使策略能够在不同几何形状、装配方向及抓取方式间泛化，并实现零样本迁移到真实环境中。

此外，作者构建了一个包含 7 类多部件装配任务的基准套件。实验表明，该系统无需人工演示或领域知识，即可完成具有代表性的真实世界多部件装配任务（步骤成功率约 80%）。

### 论文介绍

![Fabrica论文介绍图](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260417190247945.png)

#### 问题介绍

这篇论文主要解决多部件组装的问题，多部件装配不仅涉及不同的插入与抓取姿态，还需要在双臂之间频繁切换夹持—操作角色以维持整体结构稳定。

核心困难可以归纳为：

- **组合爆炸问题**：联合搜索装配顺序、抓取和运动轨迹，在离散（序列）与连续（位姿/轨迹）空间中形成大规模混合优化问题；
强耦合性：
- 装配顺序会影响抓取空间，抓取又影响运动可行性，三者相互制约；
- **接触不确定性**：插入操作必须对错位、摩擦及感知误差具有鲁棒性，同时需要跨不同几何结构泛化。

#### 方法概要

- 算法层面提出了一种分层双臂规划器，将原始联合优化问题拆解为：优先级 → 抓取 → 序列 → 运动的层级求解过程，从而显著降低搜索复杂度；
- 系统层面构建真实机器人系统，实现从 CAD 装配模型到执行的完整映射流程，在轨迹跟踪（planning）与反应式控制（learning）之间交替；
- 基准方面设计多部件装配任务（5–9 个零件），验证系统在仿真与真实环境中的一致性与泛化能力。

### 模型框架


![Fabrica模型架构图](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260416203225542.png)

#### 问题定义

这篇论文将多部件装配问题定义为**装配-夹持序列 $\phi$、抓取位姿 $\sigma$ 和机器人运动轨迹 $\pi$ 的联合优化问题**，其数学形式为：

$$
\min_{\phi,\sigma,\pi} E\Big(\Phi_{i=1}^{n} \tilde{f}(\phi_{1:i},\sigma_{i-1:i},\pi_i)\Big)
$$

$$
\text{s.t.} \quad C_{\text{prec}}(\phi) \le 0,\; C_{\text{kin}}(\phi,\sigma,\pi) = 0,\; C_{\text{col}}(\phi,\sigma,\pi) \le 0
$$

其中：

- $\phi$ 是部件装配与夹持的顺序（交替进行）；
- $\sigma$ 是每一步的抓取位姿；
- $\pi$ 包含自由移动（transit）、抓取移动（transfer）和装配插入运动；
- $\tilde{f}$ 评估每一步的质量（如稳定性、接触面积等）；
- 约束包括**装配优先级约束 $C_{\text{prec}}$**、运动学约束 $C_{\text{kin}}$ 和碰撞约束 $C_{\text{col}}$。

#### 算法流程

系统输入端严格限定为三类数据，这些配置直接定义了后续所有逆运动学求解与碰撞检测的约束边界：

- **零件网格模型**
- **机器人硬件**
- **工作单元配置**

在规划流程起点，系统首先执行**优先级规划模块**。该模块采用**反向拆卸（disassembly）策略**来推断装配顺序，其核心思想是：

> 相比直接规划装配，判断“是否可以拆卸”更容易作为几何可行性的判据。

具体过程为：从完整装配体出发，逐步检测可拆卸零件并分层移除，最终构建：

- **优先级层级（tiers）**
- **优先级有向图（partial order graph）**

该图不仅编码装配顺序约束，还存储每个零件的拆卸路径，为后续规划提供关键几何信息。


在优先级图确立后，系统进入以**抓取过滤与序列-抓取联合优化**为核心的规划链。

这里的关键设计是：

> 将昂贵的 IK + 碰撞检测从在线搜索中剥离，转化为一次性离线预计算。

系统为每个零件构建一个**可行抓取数据库（grasp feasibility database）**，其中包含：

- 单姿态可行性
- 沿拆卸路径的全轨迹可行性
- 与其他零件的碰撞元数据

这相当于构建了一个**抓取可行性 oracle**，极大减少后续搜索开销。

在此基础上，通过状态树搜索与动态规划求解最优的：

- 装配-夹持交替序列
- 对应抓取位姿

随后进入**夹具生成模块**。该模块的关键在于将复杂的 3D 抓取问题规整化：

- 将抓取方向统一映射为 **世界坐标系 -Z 方向**
- 将零件转化为“可自上抓取”的标准姿态

从而将问题转化为：

> **二维装箱（2D packing）+ 碰撞约束问题**

并结合：

- 扫掠体积检测（避免夹爪路径干涉）
- 重心投影约束（保证稳定性）

最终生成完整夹具模型。


在所有关键位姿与序列确定后，系统对各运动片段分别调用 **RRT-Connect** 进行轨迹规划。由于各阶段之间解耦，该过程可并行执行，从而显著降低整体规划时间。


#### Planning

##### 零件优先级规划

![零件优先级规划](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260417234052066.png)

论文通过**反向拆卸和优先级图构建**的方式实现零件优先级规划，具体分两步：

- 生成优先级层级

  - 从完整装配体出发，调用基于物理的拆卸运动规划器判断**哪些零件当前可以无碰撞地拆卸**。

  - 所有可拆卸的零件归入同一**优先级层级**，表示它们可以彼此独立地拆卸。

  - 将这一层零件从当前子装配中移除，对剩余零件重复上述过程，直到所有零件都被分配到某个 tier。

  - 最终得到一个**从先到后的层级序列**，先拆的 tier 在装配时必须后装，后拆的 tier 必须先装。

- 构建优先级图：该优先级图随后用于后续的序列‑抓取优化搜索，确保搜索空间仅包含物理可行的装配顺序。

##### 双臂抓取过滤

![双臂抓取过滤](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260417234133894.png)

双臂抓取过滤是一套**离线、分层递进的可复用碰撞信息生成机制**，其核心在于将双臂可行性检查从在线搜索中剥离，转为一次性预计算，并利用零件的拆卸路径建立抓取与子装配体之间的**全路径碰撞记录**。

- 首先，对于任意抓取位姿进行单姿态可行性检查，不仅要验证逆运动学是否可解、抓取闭合、张开时是否与优先级前驱零件发生硬碰撞，更关键的是它会额外扫描所有非优先级零件将碰撞关系作为软约束元数据记录在抓取对象中，这相当于为每个抓取建立了一张会与哪些零件在什么状态下冲突的查找表。
- 接着进行装配抓取可行性检查，沿该零件的拆卸路径逐位姿确保抓取在整条插入路径的**每一帧**均保持无碰撞，并将路径上所有位姿的碰撞信息聚合存储。
- 其次进行夹持抓取可行性检查，在机器人夹持某零件固定不动的情况下，模拟其他所有零件依次拆卸的过程，记录夹持臂与拆卸零件之间在任何时刻的碰撞关系。
- 最终进行抓取对过滤，利用上述元数据完成双臂耦合校验。

##### 双臂序列抓取优化

![双臂序列抓取优化](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260417234412627.png)

双臂序列抓取优化在已知优先级图和预过滤抓取集的基础上，通过状态空间搜索和动态规划求解最优的装配，夹持序列与对应抓取。

在优先级约束与抓取数据库基础上，通过搜索求解：

> **最优 task-level sequence + grasp assignment**

属于典型的：

- Task and Motion Planning 的分解实现



#### Learning 控制模块

针对接触密集的插入任务，规划方法难以处理误差与不确定性，因此引入强化学习策略。

关键设计：

- **Residual Learning（残差学习）**
  [
  a = a_{\text{plan}} + a_{\text{residual}}
  ]

- 规划提供 nominal 轨迹

- RL 学习：

  - 误差修正
  - 接触适应

此外：

- 采用**轨迹引导探索（guided exploration）**
- 输入基于局部几何与接触状态

从而实现：

> 跨对象几何与装配方向的泛化能力
