# 星海图R1机器人使用指南

> [!TIP]
> 星海图R1机器人的ROS SDK已经不再维护，需要及时更新到ROS2

## 启动机器人

启动机器人比较复杂，可以按照以下步骤逐步进行，连接机机器人可以选择[本地连接](#本地连接)或者[远程连接](#远程连接)均可。

### 本地连接

如果不需要远程连接和控制R1，请连接好显示器和键鼠，直接在显示器和键盘上操作，并保持HDMI和USB电缆连接，直接开机启动机器人等待显示器显示桌面即可,默认的用户名和密码如下。

```bash
default user: nvidia
default password: nvdia
```

### 远程连接

远程连接需要确保机器人和本地电脑处于同一网段下，可以互相ping通，获取ip地址可以通过`ifconfig`命令来获取，远程连接机器人直接使用以下命令即可。

```bash
ssh nvidia@ip # 默认密码为nvidia
```


### 启动驱动程序

在控制R1的整个过程中，您需要打开多个终端，强烈建议使用tmux，常用的指令如下：
- 创建新终端：按`Ctrl + B`再按 `C`;
- 切换终端： 按`Ctrl + B`然后按数字，数字表示终端窗口的序号；
- 左右分屏：按`Ctrl + B`然后按`%`;
- 上下分屏： 按`Ctrl + B`然后按`"`

> [!IMPORTANT]
> 严格按照顺序执行，避免节点依赖错误

#### Step 1：启动FDCAN通信

```bash
sudo ip link set dev can0 type can bitrate 1000000 dbitrate 5000000 fd on
sudo ip link set up can0  
```

#### Step 2：启动 ROS Master

建议单独占用一个 tmux 窗口。

```bash
roscore
```

#### Step 3：加载工作空间环境

> [!NOTE]
> 后续每个新终端都必须执行

```bash
source ~/work/galaxea/install/setup.bash
# 建议加入到环境变量
echo "source ~/work/galaxea/install/setup.bash" >> ~/.bashrc
```

#### Step 4：启动 HDAS 系统

hdas系统是硬件抽象层，负责设备通信桥接和传感器数据分发。

```base
source ~/work/galaxea/install/setup.bash
roslaunch HDAS hdas.launch
```

#### Step 5：启动躯干与机械臂控制程序

躯干与机械臂控制负责关节控制，轨迹跟踪和手臂运动规划。


```bash
source ~/work/galaxea/install/setup.bash
roslaunch mobiman r1_jointTrackerdemo.launch
```

#### Step 6：启动底盘控制

```bash
source ~/work/galaxea/install/setup.bash
roslaunch mobiman r1_chassis_control.launch
```

### 一键启动脚本

官方SDK还提供了一键启动脚本，执行这个命令将启动所有驱动和运控接口，具体使用示例如下。

```bash
source ~/work/galaxea/install/setup.bash
cd ~/work/galaxea/install/share/startup_config/script
./ota_script.sh boot
```

## 遥控器使用说明

遥控器用于控制机器人的躯干、底盘和紧急停止。

![R1 遥控器操作指南](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260423165328775.png)

> [!IMPORTANT]
> 重要提示：在进行任何操作之前，请确认所有拨杆开关（SWA/SWB/SWC/SWD）都拨至最上方档位，这样能使R1 处于停止状态，防止其意外运行。

在使用遥控器控制R1之前，请必须先启动 CAN 驱动程序和底盘驱动程序。

```bash

sudo ip link set up can0 
# 启动ros
roscore
# 启动HDAS
source ~/work/ginstall/setup.bash
roslaunch HDAS hdas.launch
# 启动底盘驱动控制程序
roslaunch mobiman r1_chassis_control.launch 
```

### 常见问题及解决方案

#### 躯干姿态异常

在实际使用过程中，星海图 R1 机器人出现物理姿态异常，表现为执行动作后躯干部分无法恢复至初始姿态，并在重启后将当前错误姿态作为新的初始状态，影响后续动作执行精度与稳定性。

![星海图R1机器人异常现象展示](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260423173629056.png)

机器人在动作执行过程中，受负载变化及驱动响应不一致的影响，各关节实际输出与控制指令之间存在偏差，动作执行后产生误差漂移，无法恢复正常位置，且重启后未直接归零，直接将当前姿态作为初始参考坐标。

深层次原因具体包括但不限于：

- 伺服电机重复动作可能不准确，导致躯干部位无法复原。

- 当前伺服系统可能仅基于相对位置控制，未配置编码器导致系统无法感知真实物理位置。

首先需要判断在当前硬件条件下是否有位置反馈，若初步判断机器本身不具备电机位置检测的能力，需要人工辅助的方式恢复姿态。

然后人工辅助恢复部分姿态后需要进行发布指令进行标记线对齐，先查看一下躯干的关节状态和反馈

```bash
rostopic echo /hdas/feedback_status_torso
rostopic echo /hdas/feedback_torso
```

程序可以检测到其相关位置以后然后通过发布指令进行相对位置校准至标准初始姿态，每个躯干轴需要与预定的标记线对齐，从而调整各关节至标准初始位置。

```bash
# p_des需要根据躯干四个轴实际情况去调整对应的数值，建议每次调整的数值小于0.1
rostopic pub /motion_control/control_torso_hdas_msg/motor_control \
hdas_msg/motor_control \
"header:
  seq: 0
  stamp: {secs: 0, nsecs: 0}
  frame_id: ''
name: ''
p_des: [0.0, 0.0, 0.0, 0.0]
v_des: [0.1, 0.1, 0.1, 0.1]
kp: [0.0]
kd: [0.0]
t_ff: [0.0]
mode: 0"
```
