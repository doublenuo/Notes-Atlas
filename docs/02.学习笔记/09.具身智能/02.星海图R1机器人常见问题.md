## 常见问题及解决方案

### A. 躯干姿态异常

在实际使用过程中，星海图 R1 机器人出现物理姿态异常，表现为执行动作后躯干部分无法恢复至初始姿态，并在重启后将当前错误姿态作为新的初始状态，影响后续动作执行精度与稳定性。

![星海图R1机器人异常现象展示](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260423173629056.png)

#### 问题分析

机器人在动作执行过程中，受负载变化及驱动响应不一致的影响，各关节实际输出与控制指令之间存在偏差，动作执行后产生误差漂移，无法恢复正常位置，且重启后未直接归零，直接将当前姿态作为初始参考坐标。

深层次原因具体包括但不限于：

- 伺服电机重复动作可能不准确，导致躯干部位无法复原。
- 当前伺服系统可能仅基于相对位置控制，未配置编码器导致系统无法感知真实物理位置。

#### 解决方案

首先需要判断在当前硬件条件下是否有位置反馈，若初步判断机器本身不具备电机位置检测的能力，需要人工辅助的方式恢复姿态。

人工辅助恢复部分姿态后需要进行发布指令进行标记线对齐，先查看一下躯干的关节状态和反馈

```bash
rostopic echo /hdas/feedback_status_torso
rostopic echo /hdas/feedback_torso
```

程序可以检测到其相关位置以后然后通过发布指令进行相对位置校准至标准初始姿态，每个躯干轴需要与预定的标记线对齐，从而调整各关节至标准初始位置。

```bash
# p_des需要根据躯干四个轴实际情况去调整对应的数值，建议每次调整的数值小于0.1
rostopic pub /motion_control/control_torso_hdas_msg/motor_control \
hdas_msg/motor_control \
"header:
  seq: 0
  stamp: {secs: 0, nsecs: 0}
  frame_id: ''
name: ''
p_des: [0.0, 0.0, 0.0, 0.0]
v_des: [0.1, 0.1, 0.1, 0.1]
kp: [0.0]
kd: [0.0]
t_ff: [0.0]
mode: 0"
```

![躯干1轴标记线](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260425235318759.png)

![躯干2轴标记线](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260423174647341.png)

![躯干3轴标记线](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260423174711323.png)

最后通过串口助手使用标定线标定法进行躯干零点标定，需要使用到的工具根据操作系统选择其一即可，这里以cutecom为例进行展示。
- Ubuntu系统：`sudo apt install cutecom`
- Windows系统: [友善串口调试助手](https://www.alithon.com/downloads)


使用`sudo cutecom`打开后按照下图进行设置。

![cutecom设置](https://raw.githubusercontent.com/doublenuo/ubuntu-image/main/image/20260427221831130.png)

将标定线依次插入对应的躯干接口，选择接入的设备端口并打开，发送指令`0x67 06 ff 00 00 00 00 76`按`enter`键进行发送即可。

### B. 底盘异常操作

#### 异常现象

在对底盘进行拆解检查后，未发现任何设备件存在松动或机械结构异常的情况，初步排除了机械结构问题。

进一步通过分项控制测试发现：
- **仅控制右摇杆时**：底盘运行正常，未出现异常现象
- **控制左摇杆且动作幅度较小时**：系统表现稳定，无异常
- **控制左摇杆较大幅度操作或高速运动场景下**：底盘会出现异常情况

通过多次复现测试，可以基本确认当对底盘最大移动速度进行限制后，异常现象可以被有效规避。

#### 问题分析

基于上述测试结果，当前可以做出如下判断：
- **硬件接触不良可能性较低**：已完成拆解检查，未发现松动或接触异常，异常具有条件触发性，不符合典型接触不良的随机特征
- **异常与控制幅度和速度存在明显相关性**：仅在高负载或大幅度控制输入时出现，说明问题更可能出现在功率或控制链路上
- **初步推测原因**：电机在高负载状态下可能瞬时需求较大电流，导致供电侧出现电流冲击或电压波动，进而影响控制电路稳定性，触发保护机制（如过流保护、欠压保护或驱动保护），导致底盘异常表现

#### 解决方案

针对当前问题，提出以下两类优化方案：
- 硬件层面：增加物理限位器或行程限制装置限制摇杆输入的最大幅度，避免电机进入高负载极限工况
- 软件层面：对电机控制输出进行限幅处理，对电流或PWM输出设置上限阈值，在控制逻辑中加入异常保护或限流策略

### C. Demo异常

#### 异常现象

在未对躯干部分进行零点标定（未标零）的情况下，直接运行 Demo 脚本，会导致躯干部分出现异常运动或姿态异常的问题。

该问题具有稳定复现性，在未初始化状态下执行 Demo 时必然出现。

#### 问题分析

结合当前测试情况，初步分析如下：
- **缺少零点基准导致控制失效**：Demo 脚本默认依赖于已完成零点标定的前提，未标零状态下，各关节的当前位置与逻辑坐标不一致
- **问题本质**：控制系统在未知初始位置基础上执行运动指令
